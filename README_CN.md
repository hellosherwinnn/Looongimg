**[在线演示 🚀](https://hellosherwinnn.github.io/Looongimg/)**  
> 💡 **注意**：纯前端版（FFmpeg.wasm）现已完美适配 **移动端** 浏览器（iOS/Android）。所有视频处理均在浏览器本地完成，极具隐私安全性。

## 💡 为什么做这个项目？

这个项目源于我的一个灵感。最近换了苹果手机后，发现它不像安卓手机那样原生支持方便的长截图功能。虽然苹果商店里有不少长截图软件，但在实际使用中经常会出错，拼接效果有的时候不行。因此，我想做一个能够实现“更完美”长截图效果的工具。

---

欢迎来到 **LooongImg**。本项目提供了一个从屏幕录制中拼接长截图的高效方案，涵盖了注重隐私的纯前端版和追求性能的后端版。

---

## 📂 项目结构与特性

### 1. [纯前端脱机版 (Live)](./standalone-web)
- **状态**：**已部署至 GitHub Pages**。
- **平台**：**适配桌面端与移动端**。
- **核心**：React + FFmpeg.wasm (WebAssembly)。
- **隐私**：**绝对隐私**。所有视频处理（抽帧、对齐、拼接）均在浏览器本地完成，数据无需上传。
- **安全**：实现了跨域隔离 (COI) 以开启浏览器的共享内存处理能力。

### 2. [后端核心服务版](./backend-core)
- **核心**：Node.js + Express + Native FFmpeg + Canvas。
- **性能**：利用服务器硬件优势进行高速并行处理。
- **用途**：适合 API 集成或在低配设备上处理超大/高分辨率视频。

## 🛠️ 技术实现复盘：挑战与解决方案

> [!NOTE]
> **项目状态**：当前为稳定版本。我已清理了之前的测试代码，以保持项目结构整洁。目前版本能处理绝大多数录屏，针对部分特殊场景，我后续会继续优化算法。

### 1. 纯前端版 (浏览器实现)
- **FFmpeg 加载卡死**：
    - **问题**：FFmpeg 加载后进度停在 0%。
    - **尝试**：尝试过切换版本和 CDN，最终发现是安全响应头的问题。
    - **解决方案**：引入了 `coi-serviceworker.js` 来开启 **跨域隔离 (COI)**，从而正常使用 FFmpeg 所需的共享内存。
- **生产环境下的 Worker 加载错误**：
    - **问题**：生产环境下 Web Worker 加载失败，提示 "Failed to fetch"。
    - **尝试**：尝试过教程常用的 `toBlobURL` 方案，但在 Worker 内部会导致动态导入报错。
    - **解决方案**：改为在 `/public/` 目录本地托管 **ESM** 格式的核心文件，并使用绝对路径加载，彻底解决了加载问题。

### 2. 后端核心 (算法与架构)
- **静态 UI 干扰**：
    - **问题**：固定的状态栏或导航栏会干扰拼接算法的位移计算。
    - **解决方案**：采用了 **三频带分区** 策略。将画面分为 **顶 (12%)**、**中 (主体)**、**底 (12%)** 三部分。算法只针对中间的主体部分进行位移检测，避开了顶部和底部固定 UI 的干扰。
- **并发请求处理**：
    - **问题**：并发请求会互相覆盖临时文件。
    - **解决方案**：使用 **UUID** 为每个请求创建独立的临时目录，确保 FFmpeg 和拼接逻辑在处理时互不干扰。
- **画面剧烈抖动**：
    - **问题**：画面模糊时，算法偶尔会误判出巨大的位移跳变。
    - **解决方案**：增加了 **速度限制 (Velocity Limit)** 检查。如果计算出的位移相比前几帧超过了 **25% 的高度比率**，系统会自动识别并进行校正，保证了拼接效果的平滑。

### 3. GitHub Actions (部署)
- **工作流权限 (403 错误)**：
    - **问题**：代码没问题，但部署时报 403 错误。
    - **解决方案**：发现是 GitHub 默认权限限制。我在 `.yml` 中显式添加了 `permissions: contents: write`，并在仓库设置中开启了读写权限。
- **Monorepo 路径问题**：
    - **问题**：由于网页版在子目录中，默认的构建指令失效。
    - **解决方案**：更新了 Action 配置，将构建路径指定为 `standalone-web` 目录。同时同步了 Vite 的 `base` 路径配置，解决了部署后资源 404 的问题。

## 🛠️ 核心技术栈

### 前端 (脱机版与客户端)
- **框架**：React 19, Vite 6, TypeScript
- **样式**：Tailwind CSS 4, Framer Motion (动画)
- **引擎**：**FFmpeg.wasm** (WebAssembly)
- **隔离**：`coi-serviceworker` (跨域隔离)

### 后端与基础设施 (核心服务)
- **运行环境**：**Node.js** (服务端处理)
- **API**：**Express** (Restful API)
- **原生引擎**：**Native FFmpeg** (系统级性能)
- **图像处理**：**Node-Canvas** (服务端图像合成)
- **部署**：GitHub Actions (CI/CD), GitHub Pages (托管)

### 核心算法
- **逻辑**：自研多频带共识像素匹配算法
